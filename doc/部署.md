# 部署指南

## 1. 安装 MySQL

### Windows

1. 下载 [MySQL Community Server](https://dev.mysql.com/downloads/mysql/)
2. 安装时选择 "Server only"，设置 root 密码
3. 安装完成后确认 MySQL 服务已启动

### 验证安装

```bash
mysql -u root -p
# 输入 root 密码后应进入 MySQL 命令行
```

## 2. 创建数据库和用户

```sql
-- 创建数据库
CREATE DATABASE stamp_robot CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建专用用户
CREATE USER 'stamp_robot'@'localhost' IDENTIFIED BY 'stamp_robot_pwd';

-- 授权
GRANT ALL PRIVILEGES ON stamp_robot.* TO 'stamp_robot'@'localhost';
FLUSH PRIVILEGES;
```

## 3. 安装 Python 依赖

```bash
cd MEC202
pip install -r requirements.txt
# 或使用 uv:
uv sync
```

新增依赖：`sqlalchemy`, `pymysql`, `cryptography`, `alembic`

## 4. 配置数据库连接

编辑 `config.py`，确认 MySQL 连接参数：

```python
DB_HOST     = 'localhost'
DB_PORT     = 3306
DB_USER     = 'stamp_robot'
DB_PASSWORD = 'stamp_robot_pwd'
DB_NAME     = 'stamp_robot'
```

如果修改了密码或用户名，同步修改 `alembic.ini` 中的 `sqlalchemy.url`。

## 5. 运行数据库迁移

```bash
# 创建所有表
alembic upgrade head
```

## 6. 初始化种子数据


## 7. 启动服务

```bash

make dev

# 仅后端
make dev-backend

# 仅前端
make dev-frontend
```

访问地址：http://localhost:5173

## 8. 数据库迁移管理

```bash
# 查看当前迁移状态
alembic current

# 查看迁移历史
alembic history

# 生成新的迁移脚本（修改 ORM 模型后执行）
alembic revision --autogenerate -m "描述"

# 执行迁移
alembic upgrade head

# 回退一个版本
alembic downgrade -1
```

## 常见问题

**Q: `Can't connect to MySQL server`?**
A: 检查 MySQL 服务是否启动，用户名密码是否正确。

**Q: `Access denied for user`?**
A: 确认已执行第 2 步的 GRANT 语句，密码与 config.py 一致。

**Q: `Unknown charset 'utf8mb4'`?**
A: MySQL 5.7.7+ 才支持 utf8mb4，请升级 MySQL。

**Q: 如何添加人员数据？**
```sql
USE stamp_robot;
INSERT INTO personnel (id_number, name, dept, role)
VALUES ('20210099', '你的名字', '计算机学院', 'student');
```
