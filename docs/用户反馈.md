## 二、分模块反馈

### 2.1 登录与注册

**👍 正面：**
- "登录很快，没有多余的动画，输入完就进去了。"（用户 B）
- "演示账号直接写在了登录页下方，第一次用很方便。"（用户 C）
- "注册流程简单，不需要填一堆无关信息。"（用户 D）

**🔧 改进建议：**
- "登录失败时只显示'用户名或密码错误'，不知道具体是哪个错了，有点不友好。"（用户 A）
- "能不能加个'记住我'功能？每次重新打开浏览器都要重新登录。"（用户 C）
- "注册后默认是操作员，页面上没有明确提示，我以为注册了就能当管理员。"（用户 F）

---

### 2.2 操作台盖章（通用模式）

**👍 正面：**
- "摄像头画面挺流畅的，延迟很低。"（用户 B）
- "盖章成功和失败的反馈很清晰，绿色对勾和红色叉号一目了然。"（用户 A）
- "不用手动调参数，放好文件点一下就行，对非技术人员友好。"（用户 D）

**🔧 改进建议：**
- "摄像头画面没有参考框，不知道文档放哪里是最佳位置。建议加个 A4 大小的虚线框叠在视频上。"（用户 B）
- "连续盖章时每次都要重新点按钮，能不能加个'连续模式'，换一张纸自动再拍？"（用户 A）
- "盖章过程中的声音提示只有语音播报，如果没有开声音就不知道进行到哪了。加个视觉进度条会不会更好？"（用户 C）
- "OCR 识别失败时只显示'核验未通过'，但不告诉我具体哪里不通过。比如是姓名识别错了还是日期格式不对？"（用户 D）
- "摄像头对环境光敏感，下午阳光斜射的时候识别率明显下降。能不能加个自动曝光提示？"（用户 B）

---

### 2.3 请假条盖章（SSE 流式）

**👍 正面：**
- "流式进度很直观，拍照中 → 扫描二维码 → 识别中 → 核验中 → 盖章中，每一步都能看到。"（用户 E）
- "十条核验逐条显示结果的设计很好，出了问题知道是哪一项。"（用户 F）
- "盖章前后的照片自动保存在日志里，不用手动拍照留底了。"（用户 A）

**🔧 改进建议：**
- "二维码稍微折一下或者打印模糊就扫不出来，容错性有点差。能不能用更鲁棒的二维码格式？"（用户 D）
- "请假条核验失败进入复审队列后，操作台没有任何提示告诉用户'你的文件已经提交复审'，他会以为系统卡住了。"（用户 C）
- "流式过程中如果网络波动断了，没有重连机制，只能刷新重来。建议加个自动重试。"（用户 E）
- "十条核验结果全部展示完才出结论，中间等待时间长。能不能边核验边更新结果？"（用户 F）
- "请假类型 OCR 识别偶尔会把'病假'识别成'病便'，这种同音错字会导致核验失败。"（用户 D）

---

### 2.4 请假申请管理

**👍 正面：**
- "创建申请的表格字段很全，下拉选项也很合理。"（用户 B）
- "审批后的 PDF 带二维码可以直接打印，这个闭环做得很好。"（用户 F）
- "权限控制到位，操作员确实看不到审批按钮，也看不到别人的申请。"（用户 A）

**🔧 改进建议：**
- "创建申请时提交成功只弹了一个小 toast，一闪就没了。建议跳转到一个确认页面或者至少有个显眼的提示。"（用户 C）
- "审批通过后，能不能自动发一条飞书/微信通知给申请人？否则申请人不知道已经通过了，要反复刷新页面看。"（用户 E）
- "请假列表没有搜索和筛选功能。历史申请多了以后很难找到特定的一条。"（用户 B）
- "PDF 下载的按钮不太显眼，我第一次都没找到。"（用户 D）
- "能不能批量审批？比如同一天有十几个请假申请，一个一个点很累。"（用户 F）

### 2.5 人工复审

**👍 正面：**

## 三、优先级建议

根据反馈频率和影响程度，建议优先处理：

| 优先级 | 问题 | 影响范围 | 建议方案 |
|:---:|---|------|------|
| 🔴 高 | GLM-4V API 超时无重试 | 操作员 | 添加指数退避重试，最多 3 次 |
| 🔴 高 | 复审无备注字段 | 复审员 | 加一个 textarea，审批时可选填 |
| 🔴 高 | 日志无搜索/筛选/导出 | 管理员、复审员 | 加日期筛选 + 关键词搜索 + CSV 导出 |
| 🔴 高 | 用户角色不可修改 | 管理员 | 前端加角色下拉框，开放 PUT 接口 |
| 🟡 中 | OCR 失败原因不透明 | 操作员 | 盖章失败时展示具体哪项核验未通过 |
| 🟡 中 | 请假列表无搜索筛选 | 操作员 | 加搜索框和状态筛选 |
| 🟡 中 | 摄像头无参考框 | 操作员 | 视频画面上叠加 A4 虚线框 |
| 🟡 中 | 没有"记住我" | 所有人 | 登录时可选"保持登录" |
| 🟢 低 | 深色模式 | 所有人 | TailwindCSS dark mode |
| 🟢 低 | 移动端适配 | 复审员 | 响应式布局 |
| 🟢 低 | 模板正则测试区 | 管理员 | 模板编辑页加实时预览 |
